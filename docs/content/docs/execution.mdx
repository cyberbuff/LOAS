This project provides multiple ways to execute Apple Script. You can execute it via CLI, osascript, Swift, `.app` file or run the binary directly.

The GitHub releases page provides all of these different files for each test.

- [osascript](https://github.com/loas-project/loas/releases/download/latest/osascript.zip)
- [Swift](https://github.com/loas-project/loas/releases/download/latest/swift.zip)
- [`.app` file](https://github.com/loas-project/loas/releases/download/latest/app.zip)
- [Binary](https://github.com/loas-project/loas/releases/download/latest/binaries.zip)

If you would like to compile the files yourself, you can use the following command:

```bash
git clone https://github.com/cyberbuff/loas.git
cd loas
uv sync
uv run main.py build
```

Each of these methods generate different set of logs based on the execution tool.
From what I could find in Sigma rules, Elastic Detection Rules, and Splunk Detection rules, they mostly check only for osascript CLI.

## CLI

This is the simplest way to execute commands from this repository.

```bash
osascript -e "return (the clipboard)"
```
<img alt="CLI" src="/images/osascript-cli.png" />



## osascript

Download the script from the [releases page](https://github.com/loas-project/loas/releases) and execute it with osascript.

```bash
osascript get_clipboard_content_using_applescript_defaults.scpt
```

<img alt="File" src="/images/osascript-file.png" />
## Swift

Download the Swift file from the [releases page](https://github.com/loas-project/loas/releases) and execute it with Swift.

```bash
swift get_clipboard_content_using_applescript_defaults.swift
```
<img alt="Swift" src="/images/swift.png" />
## `.app` file

Download the `.app` file from the [releases page](https://github.com/loas-project/loas/releases) and execute it.

```bash
open -n get_clipboard_content_using_applescript_defaults.app
```

<img alt="App" src="/images/app.png" />
## Binary

Download the binary from the [releases page](https://github.com/loas-project/loas/releases) and execute it.

```bash
./get_clipboard_content_using_applescript_defaults
```
